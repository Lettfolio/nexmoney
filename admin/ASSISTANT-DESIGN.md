# NexMoney Broker Assistant — Design (v1, built 9 Jul 2026)

The assistant turns the back office from a record system into a personal assistant for each broker:
it tells them what to do next, reads client emails, drafts replies, and does basic tasks on command.

## 1. My Day briefing (Today screen)

A prioritised feed at the top of Today, one row per action needed, Mine/All toggle. Generated by
Postgres RPC `get_briefing(p_scope)` — rules-based, instant, zero AI cost. Sources & priority:

| Pri | Kind | Rule |
|----|------|------|
| 10 | task_overdue | case_tasks past due, not done (mine/unassigned) |
| 12 | rate_urgent (overdue) | rate already ended, client not moved |
| 15 | lead_new | website leads awaiting accept/discard |
| 18 | email_new | untriaged inbound client emails (Outlook sync) |
| 20 | task_today | tasks due today |
| 25 | appt_today | today's appointments |
| 30 | rate_urgent | rate ends ≤60 days |
| 40 | stalled | active case, no activity 10+ days |
| 45 | fee_chase | fee requested 7+ days ago, unpaid |

Each row has one-click actions reusing existing code paths: Open case/client/appt, Done (task),
Accept lead, Queue email, Mark email handled, "Ask AI" (opens chat prefilled).

## 2. Chat assistant (drawer, all screens)

Floating button bottom-right opens a chat drawer. Backend: edge function `assistant`
(Claude claude-sonnet-5, tool-use loop, staff-only via JWT + profiles.role check). Tools the
model can use server-side: search_clients, get_case (notes/tasks/emails included), get_briefing,
create_task, add_note, queue_email (existing template types — Resend sends via process-emails).
Every write is logged to `assistant_actions` and surfaced as an action chip in the chat.
Drafting: ask it to draft any client email/reply — it returns text to copy/edit (it does not
send free-form emails; only templated email_queue types can be sent, keeping AR compliance).

## 3. Outlook email reading (Microsoft Graph)

Edge function `outlook-sync`: client-credentials flow (app-only), reads each adviser mailbox,
matches sender address against clients.email, stores matched messages in `case_emails`
(linked to client + their most recent active case). Unmatched mail is ignored — no noise.
Cursor per mailbox in `sync_state`. Triggered from the app (auto every 10 min while open, plus
a Sync button). Matched emails appear in the My Day briefing + Client emails panel with
Open case / Mark handled / Draft reply with AI.

Setup Daniel must do (one-off, ~10 min in Azure portal):
1. Entra ID → App registration "NexMoney BackOffice" → note Tenant ID + Client ID.
2. Certificates & secrets → new client secret → note value.
3. API permissions → Microsoft Graph → Application → Mail.Read → Grant admin consent.
4. Supabase → Edge Functions → Secrets: MS_TENANT_ID, MS_CLIENT_ID, MS_CLIENT_SECRET.
5. Settings screen: enable Outlook sync, confirm mailbox list.
(Recommended hardening: an application access policy in Exchange to limit the app to the
advisers' mailboxes only.)

## 4. New database objects

- `case_emails` — synced messages (graph_id unique, mailbox, client/case links, triage_status new/handled/ignored). RLS staff-only.
- `assistant_actions` — audit log of AI-performed writes. RLS staff-only.
- `sync_state` — per-mailbox sync cursors.
- RPC `get_briefing(p_scope text)` → jsonb array, security definer, staff-guarded.
- settings keys: `outlook_enabled`, `outlook_mailboxes`.

## 5. Secrets required (Supabase → Edge Functions → Secrets)

- ANTHROPIC_API_KEY — chat assistant (also unblocks ai-import / parse-offer)
- MS_TENANT_ID / MS_CLIENT_ID / MS_CLIENT_SECRET — Outlook sync
- RESEND_API_KEY — still outstanding for outbound email

## 6. Deliberate v1 limits

- Assistant sends only templated emails; free-form drafts are copy/paste (AR financial-promotion caution).
- Inbound only from Outlook (no send-as via Graph yet), matched clients only.
- No background cron for sync — runs while app is open; add pg_cron later if wanted.
